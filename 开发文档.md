# 电子人情簿开发文档

## 1. 项目简介

电子人情簿是一个本地网页应用，用于记录姓名、礼金、备忘录，支持分页展示、增删改、日志回放导入，以及 PDF / Excel 导出。

项目特点：
- 前端：`Vite + TypeScript`（原生 DOM 实现）
- 本地 API：`Express`（用于操作日志文件落盘）
- 数据存储：
  - 业务数据：`localStorage`
  - 操作日志：`data/operation-logs.json`

---

## 3. 环境要求

- Node.js 18+（建议 LTS）
- npm 9+
- Windows / macOS / Linux（当前脚本对 Windows 友好）

---

## 4. 启动方式

### 4.1 开发模式（推荐）

```bash
npm install
npm run dev
```

会同时启动：
- 前端开发服务（默认 `5173`，冲突会自动顺延）
- 日志 API（`3001`）

### 4.2 本地“应用”模式

```bash
npm run build
npm run app:start
```

默认访问：
- 网页：`http://127.0.0.1:4173`
- API：`http://localhost:3001`

### 4.3 一键启动

双击：
- `一键打开人情簿.bat`

说明：脚本会检查依赖、必要时构建、再启动服务并打开浏览器。

## 5. 核心功能说明

## 5.1 记录管理

实现文件：`src/main.ts`

- 新增记录：姓名/礼金/备忘录
- 修改记录：右侧点“修改”，左侧表单回填
- 删除记录：二次确认
- 分页：每页 10 条（`PAGE_SIZE`）

## 5.2 标题主题

实现文件：`src/main.ts` + `src/storage.ts`

- 顶部“主题标题”可编辑
- 实时显示并保存到 `localStorage`
- 导出内容与导出文件名使用该标题

## 5.3 统计

实现文件：`src/main.ts`

- 本页礼金总额（阿拉伯数字）
- 全部礼金总额（阿拉伯数字）

## 5.4 导出

- Excel：`src/export/excel.ts`
  - 每页一个工作表
  - 含标题行、表头、数据
  - 含 `礼金(大写)` + `礼金(数字)` 两列
- PDF：`src/export/pdf.ts`
  - 采用 `html2canvas + jsPDF` 防中文乱码
  - 将原始两页数据合并为一页 PDF，减少空白
  - 含标题、表格列

## 5.5 操作日志（文件落盘）

实现文件：`api/server.mjs` + `src/storage.ts`

- 日志文件：`data/operation-logs.json`
- 记录类型：
  - `add`
  - `update`
  - `delete`
  - `theme_update`
- 前端通过 API 写日志，服务端落盘 JSON。

## 6. 数据模型

定义文件：`src/types.ts`

- `GiftRecord`
  - `id`
  - `name`
  - `amount`
  - `memo`
  - `createdAt`

- `GiftOperationLog`
  - `id`
  - `type` (`add` / `update` / `delete` / `theme_update`)
  - `createdAt`
  - `recordId`
  - `before` / `after`（记录快照）
  - `beforeThemeTitle` / `afterThemeTitle`

---

## 7. 存储与导入策略

业务数据：
- `localStorage` 键：`gift-ledger-v1`

主题标题：
- `localStorage` 键：`gift-ledger-theme-v1`

操作日志：
- 文件：`data/operation-logs.json`
- API：
  - `GET /api/operation-logs`
  - `POST /api/operation-logs`

导入逻辑：
- “导入日志数据”按钮会读取日志并按时间回放，重建当前记录与主题标题。

## 8. 常见问题排查

## 8.1 访问不了网页

1. 先确认服务是否启动：
   - `npm run dev` 或 `npm run app:start`
2. 查看端口占用：
   - 前端：`5173`/`4173`
   - API：`3001`
3. 若端口冲突，先结束旧进程再重启。

## 8.2 日志文件没有新记录

1. 确保 API 在跑：`http://localhost:3001/api/health`
2. 确保不是旧进程：重启 `npm run dev`
3. 主题日志为防抖写入，输入后约 500ms 入库。

## 8.3 PDF 中文乱码

当前已使用图片化导出方案；若仍异常，优先检查浏览器字体与导出时的页面渲染环境。
